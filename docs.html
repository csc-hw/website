Getting Started

To start developing with francisco, all you need to do is link the current stylesheet:

          
            <link rel="stylesheet" href="https://cdn.rawgit.com/kjhx/francisco/v1.1/francisco.css">

That's it! All there's left to do is read the rest of the documentation.

Typography

The default typeface is determined by the browser and the operating system, helping create a more comfortable viewing experience. Using the system provided fonts often makes a webpage feel more like it fits in with other UI your device. Additionally, relying on system provided fonts vastly improves webpage performance.

All type is set in rems so that a single property, html font-size, can be used to globally modify all type and spacing. Obviously, both emphasis and strong text is supported.

Grid

Francisco uses a flexible, fully responsive three column grid system. You build up the grid by placing <div column> tags inside of a <div row>. You'll typically want to place these (or your whole webpage) inside of a<div frame>, which acts as a formatted container. This is what the standard three column grid looks like:

          <div row>
            <div column>
              <h3>column</h3>
            </div>
            <div column>
              <h3>column</h3>
            </div>
            <div column>
              <h3>column</h3>
            </div>
          </div>

We can also use the second type of column to take up two columns worth of space, creating a 60/40 split. All you have to do is replace the column attribute with double. This works in both orientations:

Navigation

You can create simple horizontal navigation with the <nav> tag. By using a <nav strong>tag, the first link will become bolder and darker than the rest of the navigation, making a title of sorts.

Additionally, you can use the <nav vertical> tag to create a vertical navigation with the same properties as the above.

Title Page One Page Two 
            
              <nav strong>
              <a href="#" title="Title">Title</a>
              <a href="#" title="Page One">Page One</a>
              <a href="#" title="Page Two">Page Two</a>
              </nav>

Code

Both inline code and complete code blocks are supported. Just wrap whatever you need in <code> tags, or wrap those in <pre> tags for a block.

Code blocks do wrap on smaller screens — this is necessary to prevent breaking the entire mobile experience.

            
              <pre>
                <code>
                  <!-- wow this is meta -->
                </code>
              </pre>
